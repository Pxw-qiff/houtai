# 客户端接口请求文档

本文档先整理客户端当前明确需要接入的四类接口：模型列表、用户积分余额、积分流水、积分充值。后续客户端新增需求时，再按同一格式补充。

## 1. 基础地址与认证

### 1.1 主系统业务接口

用于用户积分、充值、订单等业务接口。

```text
http://120.55.250.234:35174/api
```

主系统接口请求头：

```http
Authorization: Bearer <chuamgwei_token>
Content-Type: application/json
```

`chuamgwei_token` 来自主系统登录接口返回的 `data.token`。

### 1.2 登录接口

登录接口用于获取主系统业务 token，并同步返回当前用户调用 new-api 的 `newApiKey`。

基础地址：

```text
http://120.55.250.234:35174/ai-anime
```

登录请求：

```http
POST http://120.55.250.234:35174/ai-anime/user/login/login
```

请求头：

```http
Content-Type: application/json
```

请求体：

```json
{
  "loginField": "admin",
  "password": "admin123",
  "loginMode": "username",
  "machineCode": "web_1710000000000_xxxxx"
}
```

请求参数说明：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| loginField | string | 是 | 登录账号。根据 `loginMode` 可以是用户名、手机号或邮箱。 |
| password | string | 是 | 登录密码。 |
| loginMode | string | 否 | 登录方式。支持 `username`、`phone`、`email`，默认 `username`。 |
| machineCode | string | 是 | 设备标识。客户端需要固定保存，同一设备重复登录要传同一个值。 |

请求示例：

```bash
curl --location 'http://120.55.250.234:35174/ai-anime/user/login/login' \
  --header 'Content-Type: application/json' \
  --data '{"loginField":"admin","password":"admin123","loginMode":"username","machineCode":"web_test_device"}'
```

成功响应示例：

```json
{
  "success": true,
  "message": "登录成功",
  "data": {
    "token": "主系统JWT",
    "newApiKey": "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "newApiSync": {
      "success": true,
      "userId": 1,
      "tokenId": 1,
      "tokenName": "chuamgwei-default",
      "maskedKey": "sk-****abcd",
      "tokenCreated": false
    },
    "user": {
      "user_uuid": "b22915f6-ab29-4ad7-884a-c7425e226625",
      "username": "admin",
      "phone": null,
      "email": null,
      "user_permission_level": 0,
      "admin_permissions": 3,
      "user_channel_description": null,
      "expire_time": null,
      "is_trial_user": null,
      "trial_activated_at": null,
      "trial_duration_hours": null,
      "max_machine_count": 1,
      "login_count": 0
    }
  }
}
```

客户端需要保存：

| 字段 | 用途 | 建议保存位置 |
|---|---|---|
| data.token | 主系统业务接口使用，例如余额、流水、充值。 | 本地登录态。 |
| data.user.user_uuid | 当前用户 UUID。 | 本地登录态。 |
| data.user.username | 当前用户名。 | 本地登录态。 |
| data.newApiKey | 调用 new-api 网关接口使用。 | 不建议长期明文落本地，建议只在 API Key 页面展示或复制。 |

客户端本地 key 建议：

```text
chuamgwei_token
chuamgwei_user_uuid
chuamgwei_username
chuamgwei_machine_code
```

失败响应示例：

```json
{
  "success": false,
  "error": "密码错误"
}
```

常见错误：

| HTTP 状态 | error | 说明 |
|---|---|---|
| 400 | 请输入完整的登录信息 | `loginField` 或 `password` 缺失。 |
| 400 | 登录方式无效 | `loginMode` 不是 `username`、`phone`、`email`。 |
| 400 | 请提供设备标识 | `machineCode` 缺失。 |
| 401 | 用户不存在 | 账号不存在。 |
| 401 | 密码错误 | 密码错误。 |
| 403 | 账户已被封禁 | 用户被封禁。 |
| 403 | 账户已被删除 | 用户已被删除。 |
| 403 | 设备数量已达上限 | 当前账号绑定设备数量超过限制。 |

注意：

- `data.token` 只用于主系统接口，不能拿去请求 new-api 的 `/v1/chat/completions`。
- `data.newApiKey` 只用于 new-api 网关接口，请求头写法是 `Authorization: Bearer <newApiKey>`。
- 登录成功但 `newApiSync.success=false` 时，主系统登录仍然成功，但 AI 网关 key 可能不可用，客户端需要提示用户稍后重试或联系管理员。

### 1.3 new-api 网关接口

用于模型列表、OpenAI 兼容调用等。

```text
http://120.55.250.234:33000
```

new-api 网关请求头：

```http
Authorization: Bearer <newApiKey>
Content-Type: application/json
```

`newApiKey` 来自主系统登录接口返回的 `data.newApiKey`，格式通常为 `sk-xxx`。

### 1.4 主系统统一返回格式

主系统 Java 接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

客户端判断建议：

- `code === 200` 表示业务成功。
- `code === 401` 或 HTTP 401 表示登录失效，需要重新登录。
- `code !== 200` 时展示 `message`。

## 2. 查询模型列表

模型列表有两个可选入口。

### 2.1 客户端展示模型列表，推荐

用于客户端模型广场、模型选择器、模型分类展示。

```http
GET http://120.55.250.234:33000/api/client/models?category=all
```

认证：当前接口不要求登录。

请求参数：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| category | string | 否 | 模型分类。支持 `all`、`llm`、`img`、`image`、`video`、`text`。不传等同 `all`。 |

请求示例：

```bash
curl --location 'http://120.55.250.234:33000/api/client/models?category=llm'
```

响应示例：

```json
{
  "success": true,
  "data": [
    {
      "model": "gpt-4o-mini",
      "category": "llm",
      "channel_count": 2
    }
  ],
  "categories": [
    {
      "category": "llm",
      "total_models": 12,
      "total_channels": 18
    },
    {
      "category": "img",
      "total_models": 3,
      "total_channels": 5
    },
    {
      "category": "video",
      "total_models": 1,
      "total_channels": 1
    }
  ],
  "total_models": 16
}
```

字段说明：

| 字段 | 说明 |
|---|---|
| model | 模型名称，调用 AI 接口时传入 `model`。 |
| category | 模型分类。`llm` 文本模型，`img` 图片模型，`video` 视频模型。 |


响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,

    "updatedAt": "2026-03-14T10:30:00"
  }

```

字段说明：

| 字段 | 说明 |
|---|---|
| userUuid | 当前用户 UUID。 |
| totalPoints | 总积分。 |

## 4. 积分流水

### 4.1 统一积分日志，推荐接入

用于统一展示当前用户的充值、消费、退款、调账记录。客户端页面如果叫“积分流水”或“积分明细”，优先使用这个接口。

```http
GET http://120.55.250.234:35174/api/v1/credit/logs?current=1&size=20&type=CONSUME
```

兼容路径：

```http
GET http://120.55.250.234:35174/api/v1/credit/flows?current=1&size=20&type=CONSUME
```

请求头：

```http
Authorization: Bearer <chuamgwei_token>
```

请求参数：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| current | number | 否 | 当前页，默认 `1`。 |
| size | number | 否 | 每页数量，默认 `20`，最大 `100`。 |
| type | string | 否 | 日志类型：`RECHARGE` 充值、`CONSUME` 消费、`REFUND` 退款、`ADJUST` 调账。 |
| direction | string | 否 | 积分方向：`IN` 增加、`OUT` 减少。 |
| keyword | string | 否 | 按标题、业务单号、备注模糊查询。 |
| startTime | string | 否 | 开始时间，建议格式 `yyyy-MM-dd HH:mm:ss`。 |
| endTime | string | 否 | 结束时间，建议格式 `yyyy-MM-dd HH:mm:ss`。 |

请求示例：

```bash
curl --location 'http://120.55.250.234:35174/api/v1/credit/logs?current=1&size=20&type=CONSUME' \
  --header 'Authorization: Bearer <chuamgwei_token>'
```

响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [
      {
        "logId": "consume_1",
        "userUuid": "b22915f6-ab29-4ad7-884a-c7425e226625",
        "type": "CONSUME",
        "typeText": "AI 消费",
        "direction": "OUT",
        "amount": 0.001508,
        "title": "AI 对话消费",
        "status": "SETTLED",
        "statusText": "已结算",
        "bizType": "NEW_API_CHAT",
        "bizOrderNo": "req_20260314103000001",
        "balanceBefore": 9246.000268,
        "balanceAfter": 9245.998760,
        "operatorName": "new-api",
        "remark": "模型调用完成",
        "createdAt": "2026-03-14T10:30:05"
      }
    ],
    "total": 1,
    "size": 20,
    "current": 1,
    "pages": 1
  }
}
```

字段说明：

| 字段 | 说明 |
|---|---|
| logId | 日志展示 ID，前端可作为列表 key。 |
| type | 统一日志类型。 |
| typeText | 类型中文文案。 |
| direction | `IN` 表示积分增加，`OUT` 表示积分减少。 |
| amount | 展示积分数量，前端结合 `direction` 显示正负号。 |
| title | 列表展示标题。 |
| status/statusText | 业务状态和状态文案。 |
| bizType | 原始业务类型。 |
| bizOrderNo | 业务单号。 |
| balanceBefore | 变动前可用积分。 |
| balanceAfter | 变动后可用积分。 |
| operatorName | 操作来源。 |
| remark | 备注。 |
| createdAt | 日志时间。 |

客户端展示建议：

- 列表主展示：`title`、`typeText`、`amount`、`direction`、`statusText`、`createdAt`。
- `direction=IN` 显示为 `+amount`，`direction=OUT` 显示为 `-amount`。
- 筛选标签建议：全部、充值、消费、退款、调账。
- 不要让客户端传 `userUuid`，服务端只按当前登录用户查询。

### 4.2 AI 消费记录，兼容接口

该接口只返回 AI 请求聚合记录，不包含充值和调账。新页面建议使用 4.1 的统一积分日志接口。

```http
GET http://120.55.250.234:35174/api/v1/credit/consume-records?current=1&size=20
```

请求头：

```http
Authorization: Bearer <chuamgwei_token>
```

状态值：

| status | statusText | 说明 |
|---|---|---|
| PRE_CONSUMED | 结算中 | 已预扣，等待结算。 |
| SETTLED | 已结算 | 已按实际用量扣费。 |
| REFUNDED | 已退款 | 请求失败，预扣已退回。 |

## 5. 积分充值

当前充值流程：

```text
查询充值配置
  ↓
创建充值订单
  ↓
生成支付宝二维码
  ↓
客户端轮询订单状态
  ↓
订单 CREDITED 后刷新积分余额
```

### 5.5 查询充值订单状态

```http
GET http://120.55.250.234:35174/api/v1/recharge/order/{orderNo}
```

请求头：

```http
Authorization: Bearer <chuamgwei_token>
```

请求示例：

```bash
curl --location 'http://120.55.250.234:35174/api/v1/recharge/order/RC20260314103000001' \
  --header 'Authorization: Bearer <chuamgwei_token>'
```

响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "orderNo": "RC20260314103000001",
    "tradeNo": "2026031422000000000001",
    "userUuid": "b22915f6-ab29-4ad7-884a-c7425e226625",
    "amount": 10.00,
    "points": 1000.000000,
    "chargeRatio": 100.000000,
    "payType": "ALIPAY",
    "status": "CREDITED",
    "expireTime": "2026-03-14T10:45:00",
    "payTime": "2026-03-14T10:31:00",
    "creditedTime": "2026-03-14T10:31:01",
    "createdAt": "2026-03-14T10:30:00",
    "updatedAt": "2026-03-14T10:31:01"
  }
}
```

订单状态说明：

| 状态 | 说明 | 客户端处理 |
|---|---|---|
| CREATED | 本地订单已创建 | 可继续生成二维码。 |
| PRECREATING | 正在生成支付二维码 | 继续等待或稍后重试。 |
| WAIT_PAY | 等待用户付款 | 展示二维码并轮询。 |
| PRECREATE_UNKNOWN | 预下单状态未知 | 可提示重试生成二维码。 |
| PRECREATE_FAILED | 预下单失败 | 可提示重试或联系客服。 |
| PAID | 已支付，等待积分入账 | 继续轮询。 |
| CREDITED | 已入账 | 停止轮询，刷新积分余额。 |
| FAILED | 支付失败 | 停止轮询，提示失败。 |
| TIMEOUT | 支付超时 | 停止轮询，提示重新下单。 |
| CANCELLED | 已取消 | 停止轮询。 |
| EXCEPTION | 入账异常 | 停止轮询，提示联系客服。 |

轮询建议：

- 二维码展示后，每 `2 秒` 查询一次订单状态。
- 状态为 `CREDITED` 时停止轮询，并调用 `/v1/credit/balance` 刷新余额。
- 状态为 `FAILED`、`TIMEOUT`、`CANCELLED`、`EXCEPTION` 时停止轮询。
- 关闭支付弹窗时停止轮询。

| status | string | 否 | 订单状态筛选。 |

## 6. 客户端当前接入顺序建议

### 6.1 模型页

1. 调 `/api/client/models` 获取模型列表。
2. 按 `category` 分组展示。
3. `channel_count <= 0` 的模型不展示或标记不可用。

### 6.2 积分页

1. 调 `/v1/credit/balance` 展示余额。
2. 调 `/v1/credit/consume-records` 展示 AI 消费记录。
3. 后续等 `/v1/credit/flows` 补齐后，再展示完整积分明细。

### 6.3 充值页

1. 调 `/v1/recharge/settings` 展示充值配置。
2. 用户输入金额，调用 `/v1/recharge/order` 创建订单。
3. 调 `/v1/recharge/pay/alipay/precreate` 获取二维码内容。
4. 每 2 秒调用 `/v1/recharge/order/{orderNo}` 查询状态。
5. 状态为 `CREDITED` 后刷新 `/v1/credit/balance`。
